Wayne County PreK Enrollment System

Product Features Document

Note: This document contains high-level product features extracted from the specification. Features are organized in a sensible implementation order.

1. Granular Permission Management System

Short Description: Feature and function-specific permissions for all admin user types (County Admin, District Admin, School Admin, Teacher) with preset permission sets and custom configuration capabilities. Parent users have fixed permissions.

User Flow

  • Super Admin preset provides all permissions enabled for system administrators
  • County Admin preset includes county-level configuration and district management permissions
  • District Admin preset includes district-level configuration and school management permissions within their district
  • School Admin preset includes school management and teacher permissions for their assigned school
  • Teacher preset includes application processing, document verification, communication, and site visit permissions
  • Admins can select a preset when creating user accounts or create custom permission configurations
  • Custom configurations allow selecting individual permissions from logical groupings (System Management, District Management, School Management, Teacher, User Management, Data Access)
  • Higher-level users can modify permissions for lower-level users (e.g., District Admin can modify School Admin and Teacher permissions)
  • Permissions can be scoped to system-wide, district-wide, school-wide, or single school level
  • System enforces permission checks on all data access and feature usage

UI Overview

  • User creation interface with preset dropdown (Super Admin, County Admin, District Admin, School Admin, Teacher)
  • Custom permission builder with expandable permission groupings
  • Permission checkboxes organized by functional grouping (System Management, District Management, School Management, Teacher, User Management, Data Access)
  • Permission scope selector (System-wide, District-wide, School-wide, Single School)
  • Permission preview showing which features will be available with current selection
  • User management interface showing current permission configuration for each user
  • Permission modification interface for higher-level admins to edit lower-level user permissions
  • Access control indicators showing available features based on current user's permissions
  • Audit log showing permission changes and who made them

Permission Presets

  • Super Admin: All permissions enabled - complete system control
  • County Admin: System management + all district management permissions for all districts
  • District Admin: District management + all school management permissions within assigned district
  • School Admin: School management + all teacher permissions for assigned school
  • Teacher: Application processing, document verification, communication, site visit permissions for assigned school(s)

Custom Permission Examples

  • IEP Reviewer: View applicant data, view document data, verify IEP - scoped to specific schools or district
  • School Data Viewer: All data view permissions for school but no user management or modification permissions
  • Document Verification Specialist: View applicant data, view document data, verify documents, request corrections

2. Authentication System

Short Description: Flexible authentication supporting local accounts, OAuth integration, and district-specific SAML/SSO for secure identity management.

User Flow

  • Parents choose local username/password or OAuth (Google/Microsoft) during registration
  • District staff authenticate through their district's SAML/SSO provider
  • County Admin uses local authentication with optional SSO configuration
  • Password reset available for local accounts via email verification

UI Overview

  • Login screen with authentication method selection
  • Registration form for parent self-service account creation
  • Password reset flow with email verification
  • District configuration interface for SAML/SSO setup

3. Form Builder

Short Description: Dynamic form creation tool with comprehensive field types, conditional logic, validation rules, and multi-language support for building custom enrollment forms.

User Flow

  • County Admins configure the county intake form by adding optional custom fields to core required fields
  • District Admins create district-specific enrollment forms using drag-and-drop interface
  • Admins define field properties, validation rules, and conditional display logic
  • Forms support multiple languages (English, Spanish, Arabic) with RTL layout
  • Admins preview forms before publishing

UI Overview

  • Form builder interface with field palette and drag-and-drop canvas
  • Field configuration panel for setting properties and validation
  • Conditional logic builder for show/hide rules
  • Preview mode to test form behavior
  • Multi-language toggle for language-specific content

4. County Intake Form

Short Description: Pre-built standardized form with core required fields (date of birth, income, IEP status, housing status, school selection) that automatically determines eligibility and priority.

User Flow

  • Parents complete the intake form as their first step in the application process
  • System automatically calculates eligibility based on DOB, income, IEP status, and housing status
  • District custom priority questions display conditionally based on selected schools
  • Parents rank up to three school preferences
  • System displays eligibility status, priority level, and any timing restrictions upon submission

UI Overview

  • Multi-section form with progress indicator
  • Core required fields with clear validation indicators
  • School selection dropdown with ranking interface
  • Conditional fields that appear based on previous selections
  • Eligibility status summary displayed after submission

5. Automatic Eligibility Determination

Short Description: System automatically calculates age eligibility, income-based priority, and timing restrictions based on core required fields in the intake form.

User Flow

  • System evaluates date of birth to determine age eligibility and timing restrictions
  • System calculates Federal Poverty Level (FPL) percentage from income and family size
  • System assigns priority tier based on housing status, IEP verification, and FPL level
  • System applies timing restrictions (May 1, August 1, September 2) based on eligibility factors
  • Eligibility results display to parents and route to appropriate school admins for review

UI Overview

  • Eligibility status indicator on parent dashboard
  • Priority tier display with explanation of factors
  • Timing restriction notices with effective dates
  • Admin interface for reviewing flagged applications

6. Enrollment Priority System

Short Description: Multi-tiered priority system (Housing status > IEP > District custom priorities > FPL tiers) with automatic timing restrictions for enrollment holds.

User Flow

  • System automatically assigns priority based on verified factors in the specified order
  • District/School Admins verify IEP documentation and district custom priority responses
  • System applies enrollment holds based on FPL tier and timing restrictions
  • Applications progress through enrollment process based on priority and capacity availability

UI Overview

  • Priority level display on applicant records
  • Verification interface for IEP and custom priority documentation
  • Timing restriction indicators with hold dates
  • Priority-based applicant list views for admins

7. Application Lifecycle Management

Short Description: Four-stage enrollment process (Intake → Enrollment Paperwork → Enrolled → Site Visit) with specific actions, transitions, and notifications at each stage.

User Flow

  • Stage 1 (Intake): Parents submit county intake form, School Admin assigns teacher, application routes to school
  • Stage 2 (Enrollment Paperwork): Parents complete district enrollment form and upload documents, Teacher verifies all materials
  • Stage 3 (Enrolled): Teacher marks student as enrolled, assigns to classroom, student begins attending school
  • Stage 4 (Site Visit): Teacher configures calendar, Parent schedules visit, Teacher conducts ASQ assessment
  • Each stage triggers automated notifications to appropriate parties

UI Overview

  • Stage indicator showing current position in enrollment process
  • Available actions list based on current stage
  • Progress bar showing completion percentage
  • Stage-specific dashboards with relevant information and tasks

8. District Enrollment Form

Short Description: District-specific enrollment forms created by District Admins using the form builder, completed by parents immediately after intake submission.

User Flow

  • District Admins build custom enrollment forms using the form builder
  • Parents automatically proceed to district enrollment form after completing intake
  • Parents complete district-specific questions and upload required documents
  • Teachers review form completeness and verify all uploaded materials

UI Overview

  • District-specific form with custom fields and sections
  • Document upload interface with file type validation
  • Form completion status indicator
  • Teacher review interface with verification controls

9. Document Upload & Verification

Short Description: Document submission workflow where parents upload required documents and teachers verify each document with status tracking and correction requests.

User Flow

  • Parents upload documents (proof of residency, income verification, immunization records) during enrollment paperwork
  • System validates file types and converts images to PDF on mobile devices
  • Teachers review each document and mark as Verified, Needs Correction, or Missing
  • Teachers request specific corrections with explanations
  • Parents receive notifications and upload corrected documents

UI Overview

  • Document upload interface with drag-and-drop and file type validation
  • Document status dashboard showing Verified, Pending, or Action Required for each document
  • Teacher verification interface with status buttons and correction request forms
  • Notification center for document-related alerts

10. PDF Auto-Fill

Short Description: Automatic population of PDF enrollment forms with applicant data, eliminating manual data entry and reducing errors.

User Flow

  • Admins upload fillable PDF templates and map form fields to system fields
  • Teachers generate completed PDFs with one click from applicant records
  • System automatically populates PDF with applicant data including calculated fields
  • Teachers download or print completed forms for signatures and records

UI Overview

  • PDF template management interface with upload and field mapping
  • Generate PDF button on applicant records
  • Preview of populated PDF before download
  • Batch PDF generation interface for multiple applicants

11. Capacity & Waitlist Management

Short Description: School capacity tracking with automated waitlists that fill based on priority when spots become available.

User Flow

  • District Admins configure capacity for each school in their district
  • System automatically routes applications to first-choice school with available capacity
  • Applications for full schools are placed on automated waitlists
  • When spots open, system fills from waitlist based on priority order
  • System maintains 10% allocation hold for 0-400% FPL students until September 1

UI Overview

  • Capacity configuration interface for District Admins
  • Waitlist view with priority ordering and position indicators
  • Capacity status dashboard showing enrollment vs. available spots
  • Automated waitlist movement notifications

12. Teacher-Parent Communications

Short Description: Multi-channel communication system enabling teachers to contact parents via email, SMS, and logged phone calls with centralized history tracking.

User Flow

  • Teachers send emails or SMS directly from applicant records using templates or custom messages
  • Parent replies are automatically logged in communication history
  • Teachers log phone calls and in-person meetings with notes and outcomes
  • Notification badge alerts teachers to unread replies
  • System automatically tracks date of last contact for reporting

UI Overview

  • Communication panel on applicant records with email, SMS, and phone logging options
  • Email and SMS template library for quick messaging
  • Unified communication timeline showing all interactions chronologically
  • Notification badge in navigation bar for unread replies
  • Contact activity dashboard for administrators

13. Parent Portal & Status Tracking

Short Description: Real-time application status visibility for parents including current stage, available actions, document verification status, and communication history.

User Flow

  • Parents log in to view their application dashboard
  • System displays current stage, progress bar, and available actions
  • Parents view document verification status and respond to correction requests
  • Parents access communication history with teachers and school staff
  • Parents receive notifications for key events and status changes

UI Overview

  • Application dashboard with stage indicator and progress bar
  • Available actions list based on current stage
  • Document status dashboard with verification status for each document
  • Communication timeline showing all interactions
  • Notifications center for alerts and status updates

14. Site Visit Scheduling

Short Description: Teacher calendar configuration and parent self-scheduling system for home/site visits after student enrollment.

User Flow

  • Teachers configure availability windows with travel buffer times after student enrollment
  • Parents receive notification to schedule site visit after student starts attending school
  • Parents self-schedule from teacher's available time slots
  • Parents can reschedule up to 24 hours before visit
  • Teachers conduct visits and log outcomes and next steps

UI Overview

  • Teacher calendar interface with availability configuration
  • Parent scheduling interface showing available time slots
  • Visit confirmation with location details
  • Rescheduling interface with time restriction warnings
  • Post-visit logging interface for outcomes and documents

15. Ages and Stages Assessment

Short Description: ASQ questionnaire completion during site visits to assess child development and set educational goals.

User Flow

  • Teacher conducts 45-minute home visit in private space after student enrollment
  • Teacher completes Ages and Stages Questionnaire assessment during visit
  • Teacher discusses child's progress, shares curriculum, and sets goals
  • Teacher logs visit outcomes and uploads any documents collected
  • Second visit scheduled after assessment checkpoint for kindergarten transition review

UI Overview

  • ASQ questionnaire interface with assessment categories
  • Visit documentation form for outcomes and next steps
  • Document upload interface for materials collected during visit
  • Assessment history view tracking development over time

16. Classroom Assignment

Short Description: Assignment of enrolled students to specific classrooms based on age, IEP status, and program requirements with capacity management.

User Flow

  • Teachers assign fully enrolled students to classrooms based on age, IEP status, and program requirements
  • System reports classroom ratios (typically 10:1 or 12:1)
  • Teachers coordinate orientation scheduling with families
  • Parents receive classroom assignment notification with teacher contact information

UI Overview

  • Classroom assignment interface with student matching criteria
  • Capacity tracking for each classroom
  • Assignment notification preview for parents
  • Orientation scheduling interface

17. Multi-language Support

Short Description: Forms and interface available in English, Spanish, and Arabic with right-to-left layout support for Arabic.

User Flow

  • Users select preferred language during account creation or from settings
  • System displays all forms, labels, and messages in selected language
  • Admins configure language-specific content for custom fields
  • Arabic users see right-to-left layout automatically

UI Overview

  • Language selector in header or settings
  • RTL layout for Arabic language
  • Language-specific date and number formatting
  • Multi-language form preview for admins

18. Multi-child Applications

Short Description: Ability for parents to apply for multiple children with pre-filled parent/guardian data from the first application.

User Flow

  • Parents complete application for first child
  • Parents select option to add another child
  • System pre-fills parent/guardian and household data from first application
  • Parents complete child-specific information for each additional child
  • Each child has a separate application record linked to the same parent account

UI Overview

  • "Add another child" button on application completion
  • Pre-filled form fields with parent/household data
  • Child list view on parent dashboard showing all applications
  • Switch between child applications from dashboard

19. Application Transfers

Short Description: Ability to transfer applicants between schools while preserving all application data, documents, and verification status.

User Flow

  • County Admins can transfer any applicant to any school in the system
  • District Admins can transfer applicants to another school in their district if selected by parent
  • System preserves all application data, documents, and verification status during transfer
  • Receiving school admins and teachers gain access to transferred applicant

UI Overview

  • Transfer interface on applicant records for authorized admins
  • School selection dropdown for transfer destination
  • Transfer confirmation with data preservation notice
  • Transfer history log on applicant records

20. Automated Notifications

Short Description: Email and SMS notifications for key events including application submission, document requests, status changes, and site visit reminders.

User Flow

  • System automatically sends notifications based on enrollment stage transitions
  • Parents receive notifications for application received, placement, document requests, and site visits
  • Teachers receive notifications for new applications, document submissions, and communication replies
  • Admins receive notifications for capacity thresholds and system events

UI Overview

  • Notification center showing all alerts and status updates
  • Notification preferences in user settings
  • Email and SMS delivery status tracking
  • Notification templates for admins to customize messaging

21. Reporting

Short Description: Enrollment statistics, waitlist reports, demographics reporting, and contact activity metrics for administrators.

User Flow

  • County Admins view system-wide enrollment statistics and demographics
  • District Admins generate reports for their district including enrollment and waitlist status
  • School Admins access school-specific reports and classroom data
  • Admins export reports in various formats for state submissions

UI Overview

  • Report dashboard with key metrics and charts
  • Filter controls for date range, school, district, and status
  • Export options for CSV and PDF formats
  • Scheduled report generation for recurring needs

22. Data Integration

Short Description: CSV exports for manual Student Information System import with future API integration support for PowerSchool, MIStar, and Infinite Campus.

User Flow

  • District Admins export enrollment data in CSV format for manual SIS import
  • System supports field mapping configuration for different SIS formats
  • Future phase will enable direct API integration with major SIS platforms
  • Community Based Organizations utilize MiSTAR for 25-26 SY enrollment

UI Overview

  • Data export interface with filter options
  • Field mapping configuration for SIS templates
  • Export history and status tracking
  • SIS integration configuration for future API connections

23. Security & Audit Logging

Short Description: FERPA-compliant audit logging for data access, document operations, and enrollment decisions with secure data retention policies.

User Flow

  • System automatically logs all data access events with user and timestamp
  • Document uploads and downloads are tracked in audit logs
  • Enrollment decisions and status changes are logged with attribution
  • Admin configuration changes are recorded for accountability
  • Authorized admins can review audit logs for compliance and troubleshooting

UI Overview

  • Audit log viewer with search and filter capabilities
  • Event detail views showing full context of logged actions
  • Export options for audit records
  • Security incident reporting interface

24. Responsive Web Design

Short Description: Mobile-optimized responsive web application that works on all devices via browser with touch-friendly controls for common parent tasks.

User Flow

  • Users access system from any device with web browser
  • Interface automatically adapts to screen size and device type
  • Touch-friendly controls optimized for mobile interactions
  • Photo-to-PDF converter available on mobile devices for document uploads

UI Overview

  • Responsive layout that adjusts to desktop, tablet, and mobile screens
  • Touch-optimized buttons and form controls
  • Mobile-specific navigation patterns
  • Camera integration for mobile document capture
Status: Product features extracted and organized in implementation order. Features progress from foundational (authentication, roles) to core enrollment functionality (forms, eligibility, lifecycle) to supporting capabilities (communications, reporting, integration).
